# Frontend Guide

## Stack

- **Vue 3** with Composition API (`<script setup>`)
- **Inertia.js** for server-driven SPA navigation
- **shadcn-vue** for UI components (built on Reka UI headless primitives)
- **Tailwind CSS** v4 for styling
- **TypeScript** for type safety
- **@laravel/vite-plugin-wayfinder** for auto-generated route helpers
- **lucide-vue-next** for icons

## Page Patterns

### Layout

Every authenticated page follows this pattern:

```vue
<script setup lang="ts">
import { Head } from '@inertiajs/vue3';
import AppLayout from '@/layouts/AppLayout.vue';
import type { BreadcrumbItem } from '@/types';

const breadcrumbs: BreadcrumbItem[] = [
    { title: 'Dashboard', href: '/dashboard' },
    { title: 'Partners', href: '/partners' },
];
</script>

<template>
    <Head title="Partners" />
    <AppLayout :breadcrumbs="breadcrumbs">
        <!-- Page content -->
    </AppLayout>
</template>
```

### Receiving Props

Inertia passes data from Laravel controllers as Vue props:

```vue
<script setup lang="ts">
import type { PartnerOrganization, Paginated } from '@/types/partner';

const props = defineProps<{
    partners: Paginated<PartnerOrganization>;
    filters: { search?: string; category?: string };
}>();
</script>
```

### Forms

Use Inertia's `useForm` for form submissions:

```vue
<script setup lang="ts">
import { useForm } from '@inertiajs/vue3';
import guests from '@/routes/guests';

const form = useForm({
    email: '',
    redirect_url: '',
    send_email: true,
});

function submit() {
    form.post(guests.store.url());
}
</script>
```

### Navigation

Use Inertia's `router` for programmatic navigation and `Link` for links:

```vue
<script setup lang="ts">
import { router, Link } from '@inertiajs/vue3';
import partners from '@/routes/partners';

// Programmatic navigation with query params (e.g., search)
function search(query: string) {
    router.get(partners.index.url(), { search: query }, {
        preserveState: true,
        replace: true,
    });
}
</script>

<template>
    <Link :href="partners.show.url(partner.id)">
        {{ partner.display_name }}
    </Link>
</template>
```

## Route Helpers

Wayfinder auto-generates TypeScript route helpers from Laravel routes during the Vite build. They live in `resources/js/routes/` and should not be edited manually.

```typescript
import { dashboard } from '@/routes';           // dashboard.url() → '/dashboard'
import partners from '@/routes/partners';        // partners.index.url() → '/partners'
                                                 // partners.show.url(1) → '/partners/1'
                                                 // partners.create.url() → '/partners/create'
import guests from '@/routes/guests';            // guests.store.url() → '/guests'
```

Route helpers are regenerated automatically when you run `npm run dev` or `npm run build`.

## Shared Inertia Data

The following data is available on every page via `$page.props`:

```typescript
{
    name: string;              // App name
    auth: {
        user: User | null;     // Current user object
        role: string | null;   // 'admin' | 'operator' | 'viewer'
    };
    sidebarOpen: boolean;      // Sidebar state from cookie
}
```

Access in Vue:

```vue
<script setup lang="ts">
import { usePage } from '@inertiajs/vue3';
const page = usePage();
const isAdmin = page.props.auth.role === 'admin';
</script>
```

Or in templates:

```vue
<Button v-if="$page.props.auth.role !== 'viewer'">
    Add Partner
</Button>
```

## Adding shadcn-vue Components

If you need additional UI components:

```bash
npx shadcn-vue@latest add [component-name]
```

Components are installed to `resources/js/components/ui/`. See [shadcn-vue docs](https://www.shadcn-vue.com/) for the full component list.
